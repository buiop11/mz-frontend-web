# 웹/앱 기획서: 우리결정 (OurPick)

## 1. 프로젝트 개요
- **앱 이름:** 우리결정 (OurPick)
- **핵심 가치:** 커플 및 부부의 합리적이고 즐거운 의사결정을 돕는 협업 플랫폼 
                (이후 다인원 참여가능으로 확장예정) 
- **디자인 무드:** 아기자기하고 따뜻한 파스텔 톤 (pencil-new.pen 파일 참조)
- **타겟:** 혼자도 가능/결혼 준비 중인 예비 부부, 육아/살림 결정을 공유하는 부부, 데이트 코스를 고민하는 커플

---

## 2. 주요 기능 (Core Features)

### 2.1. 안건 상정 및 관리
- **안건 생성:** 예시) 유모차 구매, 청첩장 선택 등 주제별 안건 생성/ 큰카테고리 위 하위카테고리 ex) 웨딩, 식사, 구매, 데이트장소 등
- **링크 자동 파싱:** 외부 링크(청첩장, 쇼핑몰 등) 붙여넣기 시 썸네일, 가격, 제목 자동 추출 및 카드화/ 링크가 없는경우 텍스트 정보로 카드 생성 가능 
- **진행 상태 표시:** 리스트에서 '투표 중', '상대 확인 완료', '최종 결정(Pick!)' 등 직관적 상태 관리

### 2.2. 비교 및 투표 (UX/UI 중심)
- **카드 스와이프:** 후보지들을 카드 형태로 넘겨보며 비교
- **실시간 소통:** 각 후보 카드 하단에 댓글 기능 제공 (상세페이지/리스트에서 댓글 수 확인가능) 
- **의사결정 체결:** 투표 결과에 따라 "결정!" 애니메이션과 함께 최종 안건 확정 (확정 및 확정 취소시 알람) 

### 2.3. 사후 관리 및 확장성
- **공유 달력 연동:** 날짜가 포함된 안건 낙찰 시 스케줄러 자동 등록
- **체크리스트 전환:** 결정 완료된 사항은 '완료 목록'으로 이동하여 전체 준비 진척도 확인
- **확장성:** 2인(커플) 모드로 시작하여 향후 가족/모임 등 '방 초대' 형식으로 다인원 확장 가능 -> 구글 아이디로 참여 처리

## 3. 핵심 화면 설계 (Wireframe Concept)

### [화면 1] 홈: 우리의 결정 대기실
- **상단 히어로 보드:** "윤아님, 이번 주 3개의 중대 의결사항이 있어요!" (개인화 메시지)
- **중앙 카드 그리드:** 진행 중인 안건들이 동글동글한 카드로 나열
- **하단 탭 바:** 홈(결정 대기실), 로그(결정 히스토리 타임라인), **캘린더(Pick 일정)**, 카테고리(분야별 안건 관리), 마이페이지

### [화면 2] 안건 상세: 비교 및 투표
- **중앙 이미지 피드:** 파싱된 이미지와 정보가 크게 노출
- **인터랙션:** 하트(좋아요) / 댓글 / 스와이프 동작
- **투표 현황:** 커플 각각의 투표 여부를 아이콘으로 표시

### [화면 3] 히스토리: 결정의 역사 (타임라인)
- **간결한 로그:** "날짜 - 최종 낙찰 아이템 - 결정자" 위주의 심플한 세로형 타임라인
- **추억 저장:** 당시 후보군과 나누었던 짧은 대화 기록 보관

## 4. 디자인 가이드 (Ref. UI Concept)
- **Main Color:** #F2EFE9 (연한 베이지/아이보리 - 배경)
- **Sub Color:** #7EBAB5 (민트 그레이 - 헤더 및 포인트)
- **Accent Color:** #D9534F (딥레드 - 중요 버튼)
- **Typography:** 아기자기하고 가독성 좋은 라운드형 폰트 권장 / 무료 폰트사용 

### 4.1. Pencil 도안 (`pencil-new.pen`, `pencil-new-web.pen`)
- 모바일·웹 `.pen`에 위 색상 토큰과 **5탭(홈·로그·캘린더·카테고리·마이)** IA를 반영해 두었습니다. (웹은 1200폭 보드, 모바일은 390×844 · `pencil-new.pen` 하단 `17 캘린더 (하단 탭)` 프레임 추가)
- Next.js 구현 시 동일 토큰을 CSS 변수(또는 Tailwind theme)로 옮기면 설계와 코드가 맞습니다.
